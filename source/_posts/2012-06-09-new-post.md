- 例子：假定、你定義了房子物件。
	* 也許、會有人採用這樣的房子OO寫法：
	    <pre><code>
		x = new Building(); // 房子的instance
		x.openDoor(); // 房子的門打開了
		</code></pre>
		
也就是說、建立一個房子、然後房子的門打開了，這樣的寫法、其實也沒啥不對啦~你看歐
假如我有很多個房子、這樣的寫法還蠻不賴的耶~說真的…只要定義好一次Building這個class、然後、以後就可以
偷懶了、真的很好~。

那…為什摸我會覺得怪咧~這個例子也許誇張了一點啦~ XDDD 它其實、是有點…好吧~ 這摸說好了、如果它套入了以前
